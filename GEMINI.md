# WizardBot: Интеллектуальный помощник для Discord

Профессиональный инструмент для автоматизации настройки и управления Discord-серверами с использованием ИИ. Проект полностью переведен на использование DeepSeek v4 Flash через Timeweb Cloud AI. Проект построен на архитектуре специализированных агентов, способных выполнять сложные цепочки задач.

## Технологический стек
- **Язык**: Python 3.10+
- **Библиотека Discord**: `discord.py`
- **ИИ Модели**: 
  - DeepSeek v4 Flash (через Timeweb Cloud AI / OpenAI API)
- **Поиск**: `duckduckgo-search`, `beautifulsoup4`
- **База данных**: `asyncpg` (PostgreSQL)

## Архитектура системы

### 1. Агентная модель (Specialists)
Система использует иерархию специалистов:
- **Orchestrator**: Главный менеджер. Анализирует запрос пользователя, создает планы (pipelines) и делегирует задачи.
- **Infra Architect**: Создает каналы, категории, настраивает права доступа и структуру сервера.
- **User Specialist**: Управляет ролями и участниками (создание ролей, выдача, поиск пользователей).
- **Mass Action Specialist**: Выполняет массовые операции (переименование всех, удаление всех каналов и т.д.).
- **Chat Specialist**: Работает с контентом (эмбеды, вебхуки, чтение истории каналов).
- **Web Researcher**: Ищет информацию в сети для контекста (например, правила игры для настройки тематического сервера).
- **Chief Editor**: Финальный агент, который переводит технические отчеты специалистов в красивый и понятный ответ для пользователя.

### 2. Ключевые механизмы
- **Pipelines**: Возможность создания последовательных шагов выполнения с передачей данных между ними (через синтаксис `{{var_name}}`).
- **Reactive Flow**: Агенты могут ждать появления данных (например, ID созданной роли) для выполнения следующих шагов. Надежное извлечение ID реализовано в TimewebHandler.
- **Vision & Style**: Анализ существующих сообщений/каналов для подражания стилю сервера.

## Структура проекта
- `src/ai/`: Обработчик ИИ (Timeweb/DeepSeek) и определения специалистов.
- `src/bot/`: Логика Discord-бота, команды (Cogs) и UI-компоненты.
- `src/core/`: Ядро системы, менеджеры ключей, биллинга и CLI.
- `src/tools/`: Определения функций (tools), которые ИИ может вызывать для взаимодействия с Discord API.
- `main.py`: Точка входа в приложение.

## Развертывание и запуск

### Предварительные требования
Создайте файл `.env` на основе `.env.example`:
- `DISCORD_TOKEN`: Токен вашего бота.
- `TIMEWEB_API_KEY`: Ключ Timeweb Cloud (для DeepSeek).
- `ADMIN_ID`: Ваш Discord ID для доступа к админ-командам.

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск проекта
```bash
python main.py
```

### Тестирование
```bash
pytest
```

## Конвенции разработки
- **Типизация**: Обязательное использование аннотаций типов.
- **Логирование**: Использование стандартного модуля `logging`.
- **Асинхронность**: Весь код должен быть неблокирующим (async/await).
- **Безопасность**: Запрещено хардкодить ключи; все настройки только через `.env` или менеджеры конфигурации.
