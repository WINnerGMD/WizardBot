# WizardBot: Интеллектуальный помощник для Discord

[AI CONTEXT]
Это центральный файл контекста проекта. ВСЕ ИИ-АГЕНТЫ (Gemini, Claude, DeepSeek, GPT) ОБЯЗАНЫ следовать этим правилам.

## 🚨 Глобальные правила (Anti-patterns & Strict Rules)
1. **Асинхронность:** Абсолютно всё I/O должно быть `async/await`. Блокирующие вызовы убивают бота.
2. **Безопасность:** Запрет на хардкод токенов и ID. Все секреты лежат в `.env`.
3. **Идемпотентность инструментов:** Инструменты должны безопасно падать и возвращать понятные ошибки агенту через `try-except`, а не крашить всего бота.
4. **Русский язык:** При общении с пользователем или написании отчетов для пользователя всегда используется русский язык.

## 🧠 Архитектура Агентов (Specialists)
Система использует иерархию специалистов (`src/ai/specialists.py`). Один агент = одна узкая задача.
- **Orchestrator**: Главный менеджер. Анализирует запрос, создает планы (pipelines) и делегирует задачи. **Никогда** не пишет финальный текст пользователю.
- **Infra Architect**: Каналы, категории, права доступа.
- **User Specialist**: Роли и участники.
- **Mass Action Specialist**: Массовые операции.
- **Chat Specialist**: Эмбеды, сообщения, вебхуки.
- **Web Researcher**: Поиск в сети.
- **Chief Editor**: Переводит технические логи в красивый и понятный ответ для пользователя.

## ⚙️ Ключевые механизмы
- **Pipelines**: Оркестратор строит план выполнения с передачей данных между шагами через синтаксис `{{var_name}}`.
- **Reactive Flow**: Инструменты возвращают структурированные данные (`content` + `metadata`, например `{"content": "Role created", "id": 12345}`). Следующий шаг пайплайна может ждать этот `ID`. ЭТО САМАЯ ХРУПКАЯ ЧАСТЬ, не ломать возвращаемые типы!
- **Vision & Style**: Анализ стиля сервера для подражания.

## 🛠 Инфраструктура Инструментов (Tools Framework)
- Базовый класс: `BaseTool` в `src/tools/base.py`.
- Как писать новый инструмент: Наследование от `BaseTool`, реализация `async def execute`, обязательная аннотация типов и возврат словаря с данными или строки. Обязательно соблюдайте контракт возврата (словарь с `content` и `metadata`).
- Реестр: Инструменты регистрируются в `ToolRegistry` (`src/tools/base.py`).
- Определения инструментов находятся в `src/tools/definitions.py`.

## 📁 Структура проекта
- `src/ai/`: Обработчик ИИ (Timeweb/DeepSeek) и определения специалистов.
- `src/bot/`: Логика Discord-бота, команды (Cogs).
- `src/core/`: Ядро, менеджеры конфигурации.
- `src/tools/`: Функции (tools), вызываемые ИИ.
- `main.py`: Точка входа в приложение.
